# Story 1.1 — Criação da Plataforma da Mentoria

## Status

Done

## Story

**Como** aluno da Mentoria Zero-to-Hero IA,
**Quero** acessar uma plataforma online autenticada com aulas organizadas por módulo, fórum, materiais e avisos,
**Para** consumir o conteúdo da mentoria de forma organizada e segura.

**Como** administrador da plataforma,
**Quero** uma área de admin para gerenciar o conteúdo e os usuários da mentoria,
**Para** manter a plataforma atualizada e funcional sem depender de código.

## Critérios de Aceite

- [ ] AC-1: Aluno consegue se autenticar na plataforma (login com email/senha via Supabase Auth)
- [ ] AC-2: Aluno autenticado acessa a área da mentoria com aulas organizadas por módulo (cada módulo agrupa N aulas)
- [ ] AC-3: Fórum de discussão acessível para alunos autenticados (criar tópicos e responder em threads)
- [ ] AC-4: Repositório de materiais didáticos com upload via AWS S3 e link de download assinado
- [ ] AC-5: Quadro de avisos visível na área do aluno (somente admin pode publicar)
- [ ] AC-6: Página de Admin protegida por role (`admin`) com gestão de módulos, aulas, materiais e avisos
- [ ] AC-7: Rotas protegidas via React Router — não-autenticados são redirecionados para `/login`
- [ ] AC-8: Interface segue o Design System Mentoria IA (brand.html — dark theme, Space Grotesk/Mono, accent #ff6a00)

## Escopo

### IN (incluso nesta story)
- Autenticação de aluno via Supabase Auth (email + senha)
- SPA com React Router DOM (`react-router-dom`) — roteamento client-side
- Área do aluno: módulos, aulas, fórum, materiais, avisos
- Área de admin: CRUD de módulos, aulas, materiais e avisos
- Upload de materiais para AWS S3 (via API Express), download via URL assinada
- Proteção de rotas por autenticação e role
- Estrutura de navegação principal da plataforma

### OUT (fora desta story)
- Integração de pagamento (já existe via PagBank — story separada)
- Player de vídeo avançado com DRM
- Sistema de gamificação ou certificados
- App mobile
- Email transacional (boas-vindas, recuperação de senha customizada)

## Riscos

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Escopo amplo (4 módulos + admin) pode resultar em entrega parcial | Alta | Alto | Implementar fases sequencialmente; MVP mínimo = Auth + Aulas + Admin básico |
| RLS mal configurado no Supabase pode expor dados entre alunos | Média | Alto | Testar RLS explicitamente por role antes de qualquer deploy; revisar com @architect |
| AWS S3 CORS e presigned URLs podem ter comportamento inesperado em dev | Média | Médio | Configurar bucket com variáveis de ambiente separadas (dev/prod); testar upload antes de UI |

### AWS S3 — Upload de Materiais
[Source: AC-4 + Fase 1 Tasks]
- Upload via presigned URL (PUT direto do browser ao S3)
- Download via presigned GET URL com expiração (ex: 15 min)
- Env vars necessárias no server/:
  ```
  AWS_BUCKET=mentoria-ia-materials
  AWS_REGION=us-east-1
  AWS_ACCESS_KEY_ID=...
  AWS_SECRET_ACCESS_KEY=...
  ```
- Pacote: `@aws-sdk/client-s3` + `@aws-sdk/s3-request-presigner`

### Supabase RLS

- Admin: `raw_user_meta_data->>'role' = 'admin'`
- Aluno: usuário autenticado com role padrão
- RLS deve ser testado explicitamente antes de qualquer deploy
