# Story 11.3: Separacao dos workspaces Admin Mentoria e CRM Neural Hub

## Status
Done

## Contexto
A area administrativa estava misturando a gestao da plataforma de mentoria com o CRM comercial de leads. Essa mistura prejudicava a clareza operacional: administracao educacional e gestao comercial possuem objetivos, linguagem e fluxo de trabalho diferentes.

## Objetivo
Separar o CRM em um workspace proprio, mantendo a area de administracao da mentoria focada em alunos, cursos, aulas, materiais, avisos, progresso e blog.

## Criterios de Aceite
- [x] CRM disponivel em `/plataforma/crm/leads`.
- [x] Rota antiga `/plataforma/admin/leads` redireciona para `/plataforma/crm/leads`.
- [x] Menu da administracao da mentoria nao lista o CRM como item de gestao educacional.
- [x] CRM possui layout proprio com identidade "Neural Hub / CRM Comercial".
- [x] CRM mantem acesso protegido por `AdminRoute`.
- [x] CRM possui link de retorno para o Admin Mentoria.
- [x] Area da mentoria possui link separado para o CRM, sem misturar com os modulos educacionais.

## Implementacao
- Criado `CrmLayout` para o workspace comercial.
- `AdminLeadsPage` passou a usar `CrmLayout`.
- Rotas atualizadas para separar `/crm/leads` de `/admin/*`.
- Menus atualizados em `AdminLayout` e `Layout`.
