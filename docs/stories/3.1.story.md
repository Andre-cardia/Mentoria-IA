# Story 3.1 — Tipos de Aula: Schema + Admin + Frontend

## Status

InReview

## Story

**Como** mentor administrador da plataforma,
**Quero** criar aulas de diferentes tipos (vídeo, texto, atividade),
**Para** oferecer uma trilha de aprendizado diversificada aos mentorados.

## Critérios de Aceite

- [ ] AC-1: Migration adiciona `lesson_type` (enum: `video | text | activity | quiz`, default `video`) e `content` (text, nullable) à tabela `lessons` — sem quebrar linhas existentes
- [ ] AC-2: Admin consegue selecionar o tipo da aula ao criar/editar: `Vídeo`, `Texto`, `Atividade`, `Quiz`
- [ ] AC-3: Para tipo `video`, o campo `video_url` continua obrigatório; `content` é ocultado
- [ ] AC-4: Para tipo `text`, o campo `content` (textarea markdown) é obrigatório; `video_url` é ocultado
- [ ] AC-5: Para tipo `activity`, o campo `content` (textarea com instruções da tarefa) é obrigatório; `video_url` é ocultado
- [ ] AC-6: Para tipo `quiz`, ambos `video_url` e `content` são ocultados (quiz builder será na Story 3.2)
- [ ] AC-7: `LessonPage.jsx` detecta `lesson_type` e renderiza conteúdo adequado: vídeo → `LessonPlayer`; texto → markdown renderizado; atividade → instruções + campo "Descreva sua entrega" (textarea, sem submissão ao backend nesta story)
- [ ] AC-8: Botão "Marcar como concluída" funciona para todos os tipos
- [ ] AC-9: Aulas existentes (tipo `video`) não são afetadas — comportamento idêntico ao atual

## Escopo

### IN (incluso nesta story)
- Migration `20260412000008_lesson_types.sql`: ALTER TABLE lessons ADD COLUMN lesson_type + content
- Atualização de `AdminAulasPage.jsx`: seletor de tipo + campos condicionais
- Atualização de `LessonPage.jsx`: renderização condicional por tipo
- Componente `LessonTextContent.jsx`: renderiza markdown (usar `react-markdown` se já instalado ou fallback com `<pre>`)
- Componente `LessonActivityContent.jsx`: exibe instruções + textarea de entrega (sem persistência)

### OUT (fora desta story)
- Submissão e persistência de atividades no banco
- Quiz builder e player (Story 3.2)
- Q&A por aula (Story 3.3)
- Correção automática de atividades

## Dependências

- Story 1.1 ✅ — tabela `lessons` existente em produção
- Story 2.x ✅ — `AdminAulasPage.jsx` refatorada e funcional
- `react-markdown` ou similar disponível (verificar `package.json`)

## Riscos

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| ALTER TABLE com dados em produção | Baixa | Alto | `ADD COLUMN ... DEFAULT 'video' NOT NULL` — seguro para linhas existentes |
| `react-markdown` não instalado | Média | Baixo | Fallback simples com `<pre style="white-space: pre-wrap">` |
| Admin: muitos campos condicionais confundem UX | Média | Médio | Usar `select` de tipo bem visível + mostrar/ocultar campos com transição suave |

## Dev Notes

### Migration segura para produção
```sql
ALTER TABLE public.lessons
  ADD COLUMN IF NOT EXISTS lesson_type text NOT NULL DEFAULT 'video'
    CHECK (lesson_type IN ('video', 'text', 'activity', 'quiz')),
  ADD COLUMN IF NOT EXISTS content text;
```

### Renderização condicional em LessonPage
```jsx
{lesson.lesson_type === 'video' && <LessonPlayer videoUrl={lesson.video_url} />}
{lesson.lesson_type === 'text' && <LessonTextContent content={lesson.content} />}
{lesson.lesson_type === 'activity' && <LessonActivityContent content={lesson.content} />}
{lesson.lesson_type === 'quiz' && <div>Quiz disponível em breve.</div>}
```
