# Story 3.2 — Provas Objetivas (Quiz)

## Status

Ready

## Story

**Como** mentor administrador,
**Quero** criar provas objetivas de múltipla escolha vinculadas a aulas,
**Para** avaliar o conhecimento dos mentorados ao final de um conteúdo.

**Como** mentorado,
**Quero** responder provas objetivas e ver meu score imediatamente,
**Para** saber se absorvi o conteúdo e onde preciso revisar.

## Critérios de Aceite

- [ ] AC-1: Admin consegue acessar um "Quiz Builder" ao editar/criar uma aula do tipo `quiz`
- [ ] AC-2: Admin pode adicionar N questões à prova, cada uma com 2–5 opções de resposta
- [ ] AC-3: Admin marca exatamente 1 opção como correta por questão
- [ ] AC-4: Admin pode reordenar questões e opções (drag ou botões ▲▼)
- [ ] AC-5: Mentorado vê as questões uma a uma (ou todas na tela, sem rolagem excessiva) com opções de rádio
- [ ] AC-6: Ao submeter, o score é calculado no frontend (questões corretas / total × 100) e exibido
- [ ] AC-7: A tentativa é salva em `quiz_attempts` com score, respostas e timestamp
- [ ] AC-8: Mentorado pode refazer a prova (nova tentativa registrada — sem limite)
- [ ] AC-9: Mentorado vê histórico das suas tentativas (data + score) em ordem decrescente
- [ ] AC-10: Após submeter, o gabarito é exibido: quais acertou e qual era a resposta correta nas erradas

## Escopo

### IN (incluso nesta story)

## Dev Notes

### Mascarar is_correct no SELECT do aluno
**Opção 1 — RLS com policy condicional:**
```sql
-- Aluno só vê is_correct se já tem tentativa para a aula
CREATE POLICY "aluno_ve_gabarito_apos_tentativa" ON public.quiz_options
  FOR SELECT USING (
    auth.role() = 'authenticated' AND (
      -- admin vê sempre
      (auth.jwt()->'user_metadata'->>'role') = 'admin'
      OR
      -- aluno só vê is_correct após tentativa
      EXISTS (
        SELECT 1 FROM public.quiz_attempts qa
        JOIN public.quiz_questions qq ON qq.id = question_id
        WHERE qa.lesson_id = qq.lesson_id
          AND qa.user_id = auth.uid()
      )
    )
  );
```

**Opção 2 (mais simples):** Não incluir `is_correct` no SELECT do frontend até submissão — calcular comparando com os dados retornados após inserir a tentativa.

### Estrutura answers no quiz_attempts
```json
{
  "question_uuid_1": "option_uuid_a",
  "question_uuid_2": "option_uuid_c"
}
```

### Cálculo de score
```js
const correct = questions.filter(q => answers[q.id] === q.correct_option_id).length;
const score = Math.round((correct / questions.length) * 100);
```
