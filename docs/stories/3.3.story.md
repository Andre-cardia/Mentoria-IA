# Story 3.3 — Q&A por Aula

## Status

Ready

## Story

**Como** mentorado,
**Quero** fazer perguntas diretamente na aula que estou assistindo/lendo,
**Para** tirar dúvidas sobre aquele conteúdo específico sem perder o contexto.

**Como** mentor ou colega mentorado,
**Quero** responder perguntas no Q&A de uma aula,
**Para** contribuir com a comunidade de aprendizado.

## Critérios de Aceite

- [ ] AC-1: Abaixo do conteúdo principal de qualquer aula (vídeo, texto, atividade, quiz), existe uma seção "Perguntas e Respostas"
- [ ] AC-2: Qualquer usuário autenticado pode fazer uma pergunta (campo de texto + botão "Perguntar")
- [ ] AC-3: Qualquer usuário autenticado pode responder a uma pergunta existente (botão "Responder" por pergunta)
- [ ] AC-4: Perguntas e respostas são exibidas em ordem cronológica (mais antigas primeiro)
- [ ] AC-5: Cada pergunta exibe: avatar/inicial do usuário, texto, data relativa (ex: "há 2 dias")
- [ ] AC-6: Respostas ficam aninhadas abaixo da pergunta correspondente (1 nível de profundidade)
- [ ] AC-7: Admin pode marcar uma resposta como "Resposta Oficial" (badge destacado)
- [ ] AC-8: Paginação: exibir 10 perguntas por vez com botão "Ver mais"
- [ ] AC-9: Campo de nova pergunta fica desabilitado para usuários não autenticados (exibe "Faça login para participar")
- [ ] AC-10: Usuário pode excluir sua própria pergunta/resposta; admin pode excluir qualquer uma

## Riscos

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| N+1: buscar user info para cada pergunta | Alta | Médio | JOIN com profiles na query ou usar `auth.users` view — uma query com select aninhado |
| Q&A com muitas respostas aninhadas fica pesado | Baixa | Baixo | Limitar a 1 nível de aninhamento (perguntas + respostas diretas, sem sub-respostas) |
| Spam / conteúdo inapropriado | Baixa | Médio | Admin pode deletar qualquer item; futuro: moderação automática |

// Buscar perguntas (top-level) com respostas aninhadas
const { data } = await supabase
  .from('lesson_qa')
  .select(`
    id, body, is_official, created_at, user_id,
    profiles(name),
    replies:lesson_qa!parent_id(
      id, body, is_official, created_at, user_id,
      profiles(name)
    )
  `)
  .eq('lesson_id', lessonId)
  .is('parent_id', null)
  .order('created_at', { ascending: true })
