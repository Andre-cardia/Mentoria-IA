# Story 8.2 — Blog: Admin — Listagem e gestão de posts

## Status

Ready

## Story

**Como** admin,
**Quero** ver a lista de posts do blog com filtros por status e ações de gestão,
**Para que** eu possa gerenciar o conteúdo publicado, rascunhos e agendados.

## Contexto

Cria a página `AdminBlogPage` no painel admin. Lista todos os posts com status badge, visibility badge, autor, data de publicação. Permite filtrar por status e navegar para criação/edição. Botão de exclusão com confirmação.

## Acceptance Criteria

### AC1 — Listagem
- [ ] Query: `supabase.from('posts').select('id, title, slug, status, visibility, author_name, published_at, created_at').order('created_at', { ascending: false })`
- [ ] Cada item exibe: título, badge de status, badge de visibilidade, autor, data
- [ ] Badges de status: `draft`=cinza, `published`=verde, `scheduled`=azul, `archived`=vermelho apagado
- [ ] Badges de visibility: `public`=sem destaque, `private`=laranja accent

### AC2 — Filtros
- [ ] Filtro por status: "Todos", "Rascunho", "Publicado", "Agendado", "Arquivado"
- [ ] Filtro aplicado client-side (sem refetch)

### AC3 — Ações
- [ ] Botão "+ Novo Post" → navega para `/plataforma/admin/blog/novo`
- [ ] Botão "Editar" em cada item → navega para `/plataforma/admin/blog/:id/editar`
- [ ] Botão "Excluir" → `confirm()` nativo → `supabase.from('posts').delete().eq('id', id)`
- [ ] Estado vazio: "Nenhum post cadastrado."

### AC4 — Navegação e layout
- [ ] Usa `AdminLayout` (padrão das outras páginas admin)
- [ ] Link "Blog" adicionado ao sidebar do admin em `AdminLayout.jsx`
- [ ] Rota `/plataforma/admin/blog` adicionada em `plataforma-main.jsx`

### AC5 — Sem regressões
- [ ] Outras páginas admin inalteradas
- [ ] `npm test` passa (32/32)
