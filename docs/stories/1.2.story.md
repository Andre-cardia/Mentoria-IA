# Story 1.2 — Player de Vídeo e Progresso do Aluno

## Status

Ready for Review

## Story

**Como** aluno da Mentoria Zero-to-Hero IA,
**Quero** assistir às aulas com player embarcado na plataforma e marcar aulas como concluídas,
**Para** acompanhar meu progresso dentro de cada módulo e saber o que já consumi.

**Como** administrador da plataforma,
**Quero** visualizar quantas aulas cada aluno concluiu por módulo,
**Para** acompanhar o engajamento da turma sem sair da plataforma.

## Critérios de Aceite

- [ ] AC-1: Ao clicar em "Assistir", a aula abre com player embarcado (iframe YouTube/Vimeo) dentro da própria página — sem abrir nova aba
- [ ] AC-2: Abaixo do player, o aluno pode clicar em "Marcar como concluída" — ação persiste no Supabase (tabela `lesson_progress`)
- [ ] AC-3: Aulas já concluídas exibem badge visual "✓ Concluída" na lista de aulas do módulo
- [ ] AC-4: Cabeçalho de cada módulo exibe progresso "X / Y aulas concluídas" com barra de progresso
- [ ] AC-5: Progresso geral do curso exibido no topo da `ModulosPage` ("X aulas concluídas de Y total")
- [ ] AC-6: RLS da tabela `lesson_progress` garante que aluno só acessa seu próprio progresso
- [ ] AC-7: Admin consegue ver tabela de progresso por aluno em nova página `/admin/progresso`

## Escopo

### IN (incluso nesta story)
- Novo componente `LessonPlayer.jsx` — iframe embarcado (YouTube embed URL + Vimeo embed URL)
- Nova migration Supabase: tabela `lesson_progress` com RLS
- Endpoint ou lógica client-side para registrar/consultar progresso
- Badge "✓ Concluída" na lista de aulas
- Barra de progresso por módulo e total
- Página admin `/admin/progresso` — view read-only por aluno
- Testes unitários para LessonPlayer e lógica de progresso

### OUT (fora desta story)
- Tracking automático de progresso por tempo assistido (requer postMessage API do player)
- Player customizado com DRM ou streaming HLS
- Certificado de conclusão
- Notificação ao mentor quando aluno conclui módulo
- Integração com plataformas de vídeo além de YouTube e Vimeo

## Riscos

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| YouTube/Vimeo bloqueiam iframe por política de segurança (X-Frame-Options) | Baixa | Alto | Usar URLs de embed (`/embed/VIDEO_ID`) e não a URL de watch; testar com vídeos reais antes de entregar |
| RLS mal configurada permite aluno ver progresso de outro | Média | Alto | Testar explicitamente com dois usuários diferentes antes de marcar Done |
| Barra de progresso com re-render excessivo ao marcar aula | Baixa | Baixo | Usar estado local + optimistic update, sincronizar com Supabase assincronamente |

## Dev Notes

### Converter URL de vídeo para embed

```javascript
// LessonPlayer.jsx — detecção de plataforma
function toEmbedUrl(url) {
  if (!url) return null;
  // YouTube: watch?v=ID ou youtu.be/ID
  const yt = url.match(/(?:youtube\.com\/watch\?v=|youtu\.be\/)([A-Za-z0-9_-]{11})/);
  if (yt) return `https://www.youtube.com/embed/${yt[1]}`;
  // Vimeo: vimeo.com/ID
  const vi = url.match(/vimeo\.com\/(\d+)/);
  if (vi) return `https://player.vimeo.com/video/${vi[1]}`;
  return null;
}
```

### Supabase — RLS lesson_progress

```sql
-- aluno lê/gerencia apenas seu progresso
CREATE POLICY "aluno gerencia progresso" ON lesson_progress
  FOR ALL USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

-- admin lê todos
CREATE POLICY "admin lê progresso" ON lesson_progress
  FOR SELECT USING (
    (auth.jwt()->'user_metadata'->>'role') = 'admin'
  );
```
