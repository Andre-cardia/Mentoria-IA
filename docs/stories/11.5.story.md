# Story 11.5: Responsividade mobile das paginas Neural Hub

## Status
Done

## Contexto
As paginas publicas da Neural Hub, proposta e agradecimento foram criadas com foco desktop e apresentavam cortes de texto, CTAs apertados e composicoes pouco legiveis em telas menores.

## Objetivo
Ajustar as paginas publicas para funcionar bem em mobile, preservando a identidade visual Neural Hub e evitando overflow horizontal.

## Criterios de Aceite
- [x] Home Neural Hub possui header, hero, secoes, comparativos, estatisticas, fundadores, parceiros e footer adaptados para mobile.
- [x] Pagina `solicitar-proposta` possui hero, CTAs, sinais e formulario legiveis em telas pequenas.
- [x] Pagina `obrigado.html` possui header, hero, CTAs, cards e footer sem cortes horizontais.
- [x] Textos longos possuem quebra segura em mobile.
- [x] Build de producao continua funcionando.
- [x] Testes automatizados continuam passando.

## Implementacao
- Criadas regras responsivas na home Neural Hub para navegação, hero, grids, cards, comparativos, estatisticas e footer.
- Ajustada a pagina de proposta para reorganizar header, CTAs, hero e grid de sinais em mobile.
- Ajustado `obrigado.html` para reorganizar header, reduzir tipografia extrema, limitar larguras e proteger textos/CTAs contra overflow.
- Validacao visual feita com Chrome headless em viewport estreita.
