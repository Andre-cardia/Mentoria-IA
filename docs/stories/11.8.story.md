# Story 11.8: Gestao de usuarios admin e comercial

## Status
Done

## Contexto
Os acessos `admin` e `comercial` eram criados manualmente no Supabase Auth por meio de `raw_user_meta_data`. A plataforma ja possuia gestao de alunos, mas nao uma area separada para usuarios internos da administracao e do CRM.

## Objetivo
Criar uma area administrativa para cadastrar e gerenciar usuarios da plataforma com roles `admin` e `comercial`.

## Criterios de Aceite
- [x] Admin acessa uma pagina separada de usuarios da plataforma.
- [x] Admin cria usuario com nome, e-mail, senha inicial e role.
- [x] Roles disponiveis sao apenas `admin` e `comercial`.
- [x] Admin edita nome e role de usuarios internos.
- [x] Admin nao consegue remover o proprio acesso admin.
- [x] Admin nao consegue excluir o proprio usuario.
- [x] Usuarios comerciais continuam separados de alunos e aparecem na lista de responsaveis do CRM.

## Implementacao
- Criada pagina `/plataforma/admin/usuarios`.
- Adicionado item "Usuarios" no menu admin.
- Criadas APIs `/api/admin/platform-users` e `/api/admin/platform-users/[id]` usando Supabase Service Key.
- APIs gravam `user_metadata.full_name` e `user_metadata.role` no Supabase Auth.
- Adicionados testes de listagem, criacao e protecoes contra auto-remocao.
