# Story 11.4: CRM com lead manual e acesso comercial

## Status
Done

## Contexto
O CRM precisava ser operado por pessoas da area comercial sem conceder acesso administrativo completo a gestao da mentoria. Tambem faltava uma forma de cadastrar leads recebidos fora do formulario publico, como WhatsApp, ligacao, indicacao ou evento.

## Objetivo
Permitir cadastro manual de leads no CRM e criar o nivel de acesso `comercial`, restrito ao workspace CRM Neural Hub.

## Criterios de Aceite
- [x] CRM possui botao "Novo lead" para cadastro manual.
- [x] Formulario manual grava em `proposal_requests`.
- [x] Lead manual entra no Kanban sem recarregar a pagina.
- [x] Novo role `comercial` acessa `/plataforma/crm/leads`.
- [x] Role `comercial` nao acessa rotas `/plataforma/admin/*`.
- [x] Role `comercial` nao cai no fluxo de completar perfil da mentoria.
- [x] RLS libera leitura e atualizacao do CRM para `admin` e `comercial`.

## Implementacao
- Criado `CrmRoute` com permissao `admin` ou `comercial`.
- `AuthContext` passou a expor `role`, `isCommercial` e `hasCrmAccess`.
- Login redireciona `role=comercial` para `/crm/leads`.
- `AdminLeadsPage` recebeu modal de cadastro manual.
- Migration adicionada para policies de CRM com `role in ('admin', 'comercial')`.

## Como criar um usuario comercial
No Supabase SQL Editor, aplicar no usuario desejado:

```sql
update auth.users
set raw_user_meta_data = coalesce(raw_user_meta_data, '{}'::jsonb) || '{"role": "comercial"}'::jsonb
where email = 'comercial@empresa.com.br';
```
