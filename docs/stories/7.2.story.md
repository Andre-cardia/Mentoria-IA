# Story 7.2 — Materiais: Admin seletor de curso no upload

## Status

Ready

## Story

**Como** admin,
**Quero** selecionar a qual curso pertence um material durante o upload (ou indicar que é "Conteúdo Extra"),
**Para que** os materiais sejam organizados corretamente para os alunos.

## Contexto

Continuação da Story 7.1 (coluna `module_id` já existe). Esta story adiciona o seletor de curso ao formulário de upload do admin, atualiza a API para persistir o `module_id`, e exibe um badge de curso na lista de materiais.

## Acceptance Criteria

### AC1 — Formulário de upload
- [ ] Dropdown "Curso" adicionado ao formulário, após o campo "Descrição"
- [ ] Opções do dropdown: primeira opção é "Conteúdo Extra" (valor vazio/null) + um item por curso em `modules` ordenado por `order ASC`
- [ ] Dropdown carregado via `supabase.from('modules').select('id, title').order('order', { ascending: true })`
- [ ] Seleção padrão: "Conteúdo Extra"
- [ ] Campo não é obrigatório (submit funciona sem selecionar curso)

### AC2 — API `POST /api/materials/upload`
- [ ] Aceita campo opcional `module_id` no body
- [ ] Insere `module_id` (ou `null` se vazio/omitido) na tabela `materials`

### AC3 — Lista de materiais no admin
- [ ] Query de materiais atualizada: `supabase.from('materials').select('*, modules(title)').order('created_at', { ascending: false })`
- [ ] Cada item exibe badge com o nome do curso se `module_id` preenchido
- [ ] Materiais sem curso exibem badge "Conteúdo Extra"

### AC4 — Toasts e UX
- [ ] Formulário reseta `module_id` para `''` após upload bem-sucedido
- [ ] Nenhum `alert()` ou `confirm()` nativo (exceto para exclusão já existente)

### AC5 — Sem regressões
- [ ] Upload sem seleção de curso continua funcionando (null)
- [ ] `npm test` passa (32/32)

## Escopo Técnico

### Arquivos a Modificar
- `src/plataforma/pages/admin/AdminMateriaisPage.jsx` — dropdown + badge na lista
- `api/materials/upload.js` — aceitar e persistir `module_id`

### NÃO fazer nesta story
- Agrupar materiais por curso na área do aluno (Story 7.3)
- Seção de materiais na CursoPage (Story 7.4)
- Edição/troca de curso de um material já enviado

### Padrões
- Inline styles existentes — não criar CSS externo
- `supabase` client direto no frontend; service key na API
- Toast via `sonner` para feedbacks de sucesso/erro

### Query — Carregar cursos para o dropdown
```javascript
supabase.from('modules').select('id, title').order('order', { ascending: true })
```

### API — body esperado
```json
{
  "fileName": "arquivo.pdf",
  "contentType": "application/pdf",
  "title": "Título do material",
  "description": "Opcional",
  "module_id": "uuid-do-modulo-ou-omitido"
}
```

## Tarefas de Implementação

- [ ] Carregar lista de cursos no mount do componente
- [ ] Adicionar campo `module_id: ''` ao estado `form`
- [ ] Renderizar dropdown de curso no formulário
- [ ] Passar `module_id` no body do fetch para `/api/materials/upload`
- [ ] Atualizar `api/materials/upload.js` para receber e inserir `module_id`
- [ ] Atualizar query de listagem para incluir `modules(title)`
- [ ] Renderizar badge de curso em cada item da lista
