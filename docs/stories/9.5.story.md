# Story 9.5 — Migration `featured` + toggle no admin

## Status

InProgress

## Story

**Como** admin do blog,
**Quero** marcar um post como "destaque",
**Para que** ele apareca em posicao privilegiada na pagina do blog.

## Contexto

Adiciona coluna `featured` na tabela `posts` do Supabase e um toggle no `AdminBlogEditorPage` para controlar qual post aparece no hero. Apenas 1 post deve ser featured por vez (toggle desativa os demais automaticamente).

## Acceptance Criteria

### AC1 — Migration Supabase
- [x] Criar migration: `ALTER TABLE posts ADD COLUMN featured boolean NOT NULL DEFAULT false`
- [x] Criar index: `CREATE INDEX idx_posts_featured ON posts(featured) WHERE featured = true`
- [x] RLS: coluna segue as policies existentes da tabela `posts`

### AC2 — Toggle no AdminBlogEditorPage
- [x] Adicionar switch/checkbox "Destacar no Blog" no `AdminBlogEditorPage.jsx`
- [x] Posicionar no bloco Status/Visibilidade/Data (mesma row)
- [x] Estilo consistente com os selects existentes

### AC3 — Logica de exclusividade
- [x] Ao salvar post com `featured=true`: primeiro `UPDATE posts SET featured=false WHERE featured=true`, depois salvar o post
- [x] Garantir atomicidade (ambas operacoes na mesma transacao ou sequenciais)
- [x] Se post esta em status `draft`, avisar que "Post precisa estar publicado para ser destaque"

### AC4 — Incluir campo no payload de save
- [x] Adicionar `featured` ao state do form (default `false`)
- [x] Incluir `featured` no payload do INSERT/UPDATE
- [x] Carregar `featured` ao editar post existente

## Scope

### IN
- Migration Supabase
- Toggle no admin editor
- Logica de exclusividade

### OUT
- Renderizacao do FeaturedPost na BlogPage (Story 9.6)
