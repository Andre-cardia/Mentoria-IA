# Story 10.1 — Homepage Neural Hub — Estrutura + Layout aifounder.one

## Status

**Atual:** Done

## Descrição

Criar a homepage pública da Neural Hub em `neuralhub.ia.br` replicando a sequência de seções, hierarquia tipográfica, espaçamentos e padrão visual do site `https://aifounder.one/`, adaptando o conteúdo para a Neural Hub e aplicando os tokens de design do brand guide (`https://mentoria.neuralhub.ia.br/brand.html`).

A página é um novo entry point Vite dentro do projeto existente `Mentoria-IA`, sem criar um repositório separado.

## Valor de Negócio

Posiciona a Neural Hub institucionalmente no domínio principal `neuralhub.ia.br`, separando a identidade corporativa (treinamentos B2B) da landing de vendas da Mentoria (produto B2C).

### Roteamento Vercel (`vercel.json`)

Problema: o Vercel serve arquivos estáticos **antes** de processar `rewrites`. O `index.html` da Mentoria era servido para `/` mesmo com a rewrite de host. Solução: usar `redirects` (que rodam antes do filesystem) para o path raiz, e `rewrites` para todos os outros paths.

| Regra | Tipo | Efeito |
| --- | --- | --- |
| `redirects /` com `host=neuralhub.ia.br` | redirect 302 | Redireciona raiz para `/neuralhub.html` |
| `rewrites /(.*)` com `host=neuralhub.ia.br` | rewrite | Serve `neuralhub.html` para todos os outros paths |

**Limitação conhecida:** ao acessar `https://neuralhub.ia.br` (raiz) a URL muda para `https://neuralhub.ia.br/neuralhub.html` no browser. Funcional mas não ideal. Solução definitiva (Story futura): mover `mentoria` para entry point não-root e usar `neuralhub.html` como `index.html` do build.

### Tentativas que não funcionaram

| Tentativa | Motivo da falha |
| --- | --- |
| Rewrite `/(.*)`  sozinha com `has: host` | `/(.*)`  não captura o path raiz `/` no Vercel |
| Rewrite `/` + `/(.*)`  ambas com `has: host` | Rewrites rodam após filesystem — `index.html` servido antes |
| `beforeFiles`/`afterFiles` no `vercel.json` | Sintaxe do Next.js, não do Vercel estático — schema validation error |
| Edge Middleware com `@vercel/edge` | `request.nextUrl` é API do Next.js, não disponível em projetos Vite |
