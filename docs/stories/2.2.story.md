# Story 2.2 — Drag-and-Drop para Reordenação

## Status

Ready for Review

## Story

**Como** administrador da plataforma,
**Quero** reordenar módulos e aulas arrastando e soltando,
**Para que** eu possa organizar o currículo visualmente sem digitar números de ordem.

## Contexto

**Pré-requisito:** Story 2.1 deve estar Done antes desta começar.

Arquivo alvo: [AdminModulosPage.jsx](../../src/plataforma/pages/admin/AdminModulosPage.jsx)

**Problema atual:** A ordenação exige que o admin digite manualmente o campo "Ordem" (número inteiro) para cada módulo e aula. Isso é lento, propenso a erros e não-intuitivo. Com drag-and-drop, o admin arrasta o item para a posição desejada e a ordem é persistida automaticamente.

## Acceptance Criteria

### AC1 — Drag-and-Drop de Módulos

- [x] Módulos têm um handle visual de drag (ícone `⠿` na lateral esquerda)
- [x] Admin consegue arrastar um módulo e reposicioná-lo na lista
- [x] Durante o drag, o item arrastado tem opacity reduzida (0.45) e borda accent
- [x] Ao soltar, a nova ordem é refletida imediatamente na UI (optimistic update)
- [x] A ordem é persistida no Supabase via `Promise.all` para todos os módulos afetados
- [x] Se a persistência falhar, a ordem é revertida na UI com toast de erro

### AC2 — Drag-and-Drop de Aulas dentro do Módulo

- [x] Aulas dentro de um módulo aberto têm handle visual de drag
- [x] Admin consegue arrastar uma aula e reposicioná-la dentro do mesmo módulo
- [x] Ao soltar, a nova ordem é refletida imediatamente
- [x] A ordem é persistida no Supabase para todas as aulas afetadas do módulo
- [x] Se falhar, reverter com toast de erro

### AC3 — Campo "Ordem" Removido dos Forms

- [x] O campo numérico "Ordem" é removido do form de criação de módulo
- [x] O campo numérico "Ordem" é removido do form de edição de módulo
- [x] O campo numérico "Ordem" é removido do form de criação de aula
- [x] Novos módulos criados recebem `order = modules.length` automaticamente
- [x] Novas aulas criadas recebem `order = lessons[moduleId].length` automaticamente

### AC4 — Feedback Visual

- [x] Handle de drag muda o cursor para `grab` no hover
- [x] Durante o drag, cursor muda para `grabbing`
- [x] Toast de sucesso após persistir nova ordem (disparado após drop)
- [x] O handle de drag não interfere com o clique para expandir/colapsar o módulo

### Padrão de Implementação dnd-kit

```jsx
// Estrutura base para módulos
import { DndContext, closestCenter } from '@dnd-kit/core';
import { SortableContext, verticalListSortingStrategy, useSortable, arrayMove } from '@dnd-kit/sortable';
import { CSS } from '@dnd-kit/utilities';

// Hook por item
function useSortableItem(id) {
  const { attributes, listeners, setNodeRef, transform, transition, isDragging } = useSortable({ id });
  const style = {
    transform: CSS.Transform.toString(transform),
    transition,
    opacity: isDragging ? 0.5 : 1,
  };
  return { ref: setNodeRef, style, handleProps: { ...attributes, ...listeners } };
}
```

### Algoritmo de Persistência de Ordem

Após `handleDragEnd`:
1. `arrayMove` para reordenar o array local (optimistic update)
2. Mapear novo array para `{ id, order: index }`
3. Usar `Promise.all` para atualizar todos no Supabase em paralelo
4. Se qualquer `error`, reverter ao estado anterior e mostrar toast de erro
