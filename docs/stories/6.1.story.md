# Story 6.1 — Curso: DB subtitle + admin labels

## Status

Done

## Story

**Como** admin,
**Quero** cadastrar um subtítulo para cada curso e ver os rótulos "Curso/Cursos" no painel admin,
**Para que** os alunos tenham contexto extra sobre o conteúdo antes de iniciar.

## Contexto

Primeira story do Epic 6. Prepara a base de dados e a interface admin para o renomeamento de "módulo" → "curso" e para o novo campo `subtitle`. Não altera tabelas nem colunas existentes — apenas adiciona `subtitle` e ajusta labels de UI.

**Tabela alvo:** `public.modules`
**Campo novo:** `subtitle text` (nullable, sem NOT NULL)

## Acceptance Criteria

### AC1 — Migration
- [ ] Migration `20260413000018_modules_subtitle.sql` criada em `supabase/migrations/`
- [ ] SQL: `ALTER TABLE modules ADD COLUMN IF NOT EXISTS subtitle text;`
- [ ] Migration aplicada no remoto via `supabase db push`

### AC2 — AdminModulosPage — campo subtítulo
- [ ] Formulário de criação/edição de módulo (agora chamado "Curso" na UI) exibe campo "Subtítulo"
- [ ] Campo subtítulo é opcional (sem validação obrigatória)
- [ ] `EMPTY_MOD` atualizado: `{ title: '', subtitle: '', description: '' }`
- [ ] Operações INSERT e UPDATE incluem `subtitle` no payload

### AC3 — Labels admin: "módulo" → "curso"
- [ ] Título da página admin: "Módulos & Aulas" → "Cursos & Aulas"
- [ ] Botão de criação: "Novo Módulo" → "Novo Curso"
- [ ] Labels internos no formulário: "Módulo" → "Curso"
- [ ] Toasts: "Módulo criado/editado/excluído" → "Curso criado/editado/excluído"
- [ ] `AdminLayout.jsx` sidebar admin: link "Módulos & Aulas" → "Cursos & Aulas" (só label, rota `/admin/modulos` mantida)
- [ ] `Layout.jsx` sidebar aluno: "Módulos & Aulas" → "Cursos & Aulas" (só label, rota `/modulos` mantida por ora — será atualizada na Story 6.2)

### AC4 — Sem regressões
- [ ] Todas as operações de CRUD de cursos e aulas continuam funcionando
- [ ] `npm test` passa (32/32)
