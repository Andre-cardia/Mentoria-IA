# Story 5.1 — Minha Conta: Página base + edição de dados pessoais

## Status

Done

## Story

**Como** aluno autenticado,
**Quero** acessar uma página "Minha Conta" onde posso editar meu nome e telefone,
**Para que** eu possa manter meus dados pessoais atualizados sem depender do suporte.

## Contexto

O aluno hoje não tem nenhum mecanismo para editar os dados que preencheu em `CompletarPerfilPage`. Esta story cria a fundação da área de perfil — a página `/minha-conta` — com edição inline de `full_name` e `phone` (tabela `profiles`). O e-mail é exibido como read-only (editável na Story 5.3). A foto de perfil fica como placeholder (Story 5.2).

**Padrões a seguir:**
- Inline forms sem formulário global (ver `AdminModulosPage.jsx`)
- Toast via `sonner` já instalado (ver `AdminLayout.jsx`)
- `ProtectedRoute` + `ProfileGuard` (ver `/modulos` em `plataforma-main.jsx`)
- Inline styles com `var(--token)` do design system (ver `Layout.jsx`)

## Acceptance Criteria

### AC1 — Rota e acesso
- [ ] Rota `/minha-conta` criada em `plataforma-main.jsx` com `ProtectedRoute` + `ProfileGuard`

### AC2 — Layout da página
- [ ] Página usa `Layout.jsx` (sidebar + main content)
- [ ] Título da página: "Minha Conta"
- [ ] Seção "Dados Pessoais" visível com os dados atuais do usuário carregados do Supabase
- [ ] Avatar placeholder circular (40px) exibido — sem funcionalidade de upload ainda

### AC3 — Edição de dados pessoais
- [ ] Campo "Nome completo" (`full_name`) editável inline
- [ ] Campo "Telefone" (`phone`) editável inline
- [ ] Campo "E-mail" exibido como read-only (texto + ícone de cadeado ou label "(somente leitura)")
- [ ] Botão "Salvar alterações" chama `UPDATE` na tabela `profiles` via Supabase
- [ ] Botão "Cancelar" descarta alterações e volta aos valores originais
- [ ] Toast de sucesso verde ao salvar com êxito
- [ ] Toast de erro vermelho se o UPDATE falhar

### AC4 — Estado de carregamento
- [ ] Dados do perfil carregados ao montar a página via `SELECT` em `profiles WHERE user_id = auth.uid()`
- [ ] Botão "Salvar" fica desabilitado e exibe "Salvando..." durante a chamada ao Supabase
