# Story 2.4 — Editor Rich Text (TipTap) para Conteúdo de Aulas

## Status

**Atual:** InProgress
**Próximo:** InReview (após @qa gate)

## Descrição

O campo de conteúdo de aulas (`lesson_type: text` e `lesson_type: activity`) usa atualmente um `<textarea>` simples em `AdminModulosPage.jsx`. O conteúdo é salvo como texto plano em `lessons.content` (TEXT no Supabase) e renderizado no player com `<pre>` sem parse de HTML.

Isso impede o admin de formatar o conteúdo de forma rica (negrito, títulos, listas, links, imagens), resultando em aulas pouco profissionais para o aluno.

O projeto **já possui TipTap v3.22.3 instalado e funcional** no editor de blog (`AdminBlogEditorPage.jsx`). Esta story extrai essa lógica para um componente reutilizável e o integra no formulário de aulas (tanto no form de nova aula quanto no formulário de edição inline), além de atualizar o player (`LessonTextContent.jsx`) para renderizar o HTML gerado pelo TipTap.

## Valor de Negócio

Admin consegue criar aulas ricas em formatação (negrito, listas, imagens, links, títulos), resultando em melhor experiência de aprendizado para o aluno. Elimina o uso de texto plano/pre-wrap no player.

## Acceptance Criteria

- [x] AC-1: Ao selecionar `lesson_type = text` ou `activity` no form de nova aula, o campo de conteúdo exibe um editor TipTap com toolbar (B, I, H1, H2, H3, listas, link, imagem, blockquote) — idêntico ao do blog
- [x] AC-2: Ao editar inline uma aula existente de tipo `text` ou `activity`, o campo de conteúdo também exibe o editor TipTap com o conteúdo carregado corretamente
- [x] AC-3: O conteúdo é salvo no Supabase como **HTML string** (`editor.getHTML()`) no campo `lessons.content`
- [x] AC-4: O componente `LessonTextContent.jsx` renderiza o conteúdo usando `dangerouslySetInnerHTML` com os mesmos estilos CSS do TipTap do blog (headings, listas, links, imagens, blockquotes)
- [x] AC-5: Aulas existentes com conteúdo em texto plano continuam renderizando sem quebrar (o `<pre>` original pode ser mantido como fallback baseado em detecção de HTML)
- [x] AC-6: O modal de inserção de imagem funciona no editor de aulas usando o mesmo endpoint `/api/blog/image-upload`
- [x] AC-7: O componente `LessonRichEditor` é extraído como componente reutilizável em `src/plataforma/components/LessonRichEditor.jsx` — não duplicar lógica diretamente em `AdminModulosPage`
- [ ] AC-8: Nenhuma regressão nas funcionalidades existentes de `AdminModulosPage` (drag-and-drop, duplicar, confirmar exclusão, toast, etc.)

## Escopo

### IN — Incluído nesta Story

- Criar `src/plataforma/components/LessonRichEditor.jsx` com TipTap + toolbar + ImageModal (extraídos de `AdminBlogEditorPage.jsx`)

- Integrar `LessonRichEditor` no form de **edição inline** de aula em `AdminModulosPage.jsx`
- Atualizar `LessonTextContent.jsx` para renderizar HTML via `dangerouslySetInnerHTML`

- Novo endpoint de upload de imagem dedicado para aulas (reutiliza `/api/blog/image-upload`)
- Sanitização de HTML no backend (fora do escopo desta story)

### Padrão de save

O blog salva `editor.getJSON()` (JSON). **Para as aulas, usar `editor.getHTML()`** — string HTML diretamente no campo `content`. Isso simplifica a renderização no player sem precisar do pacote `@tiptap/html` para conversão.

## Riscos

| Risco | Probabilidade | Mitigação |
|-------|--------------|-----------|
| `useEditor` re-criado a cada render ao passar `value` como `content` | Alta | Usar `useEffect` + `editor.commands.setContent()` para init |
| ImageModal em modal inline de edição quebrar z-index | Média | Adicionar `zIndex: 9999` no overlay do ImageModal |
| Conteúdo legado (texto plano) quebrar no player | Baixa | Detecção `content.startsWith('<')` como fallback |
| Editor em form novo não limpar após salvar | Média | Chamar `editor.commands.clearContent()` após save bem-sucedido — expor via ref ou callback |
