# Story 5.2 — Minha Conta: Foto de perfil (upload S3 + exibição no header)

## Status

Done

## Story

**Como** aluno autenticado,
**Quero** fazer upload de uma foto de perfil e vê-la exibida no sidebar,
**Para que** minha experiência na plataforma seja mais personalizada e reconhecível.

## Contexto

Continuação da Story 5.1. Adiciona a funcionalidade de foto de perfil usando o mesmo padrão de upload já implementado para materiais (`AdminMateriaisPage.jsx` + `server/routes/materials.js` + `server/adapters/s3-adapter.js`).

**Decisões de arquitetura (@architect, 2026-04-13):**
- Storage: **AWS S3**, reutilizando `S3Adapter` existente
- Bucket path: `avatars/{user_id}.{ext}` — sobrescreve ao trocar foto
- Formatos aceitos: JPEG, PNG, WebP
- Tamanho máximo: **2 MB** (validação client-side antes do upload)
- Resize: **CSS only** — `object-fit: cover` no container circular

**Padrão de upload a seguir** (idêntico ao de materiais):
1. Cliente → `POST /api/profile/avatar-upload` (body: `{ fileName, contentType }`) → recebe `{ uploadUrl }`
2. Cliente → `PUT {uploadUrl}` (upload direto ao S3)
3. Cliente → `PATCH /api/profile` com `{ avatar_url }` → persiste no Supabase

## Acceptance Criteria

### AC1 — Migration de banco
- [ ] Migration `ALTER TABLE profiles ADD COLUMN avatar_url text` aplicada via `supabase/migrations/`
- [ ] Campo `avatar_url` sem `NOT NULL` (null = sem foto)

### AC2 — Rota servidor: presigned URL
- [ ] `POST /api/profile/avatar-upload` criada em `server/routes/profile.js`
- [ ] Requer Bearer token válido (aluno autenticado)
- [ ] Aceita body `{ fileName, contentType }`
- [ ] Valida `contentType` — permite apenas `image/jpeg`, `image/png`, `image/webp`
- [ ] Gera presigned URL via `S3Adapter.getUploadUrl()` com path `avatars/{user_id}.{ext}`
- [ ] Retorna `{ uploadUrl, avatarUrl }` — `avatarUrl` é a URL pública final no S3

### AC3 — Rota servidor: atualizar perfil
- [ ] `PATCH /api/profile` criada em `server/routes/profile.js`
- [ ] Aceita body `{ avatar_url }` (e futuramente outros campos)
- [ ] Atualiza tabela `profiles` via Supabase service key
- [ ] Retorna `{ success: true }`

### AC4 — Upload no frontend (MinhaContaPage)
- [ ] Botão/área de clique sobre o avatar placeholder para selecionar arquivo
- [ ] `accept="image/jpeg,image/png,image/webp"` no input file
- [ ] Validação client-side: rejeitar arquivos > 2 MB com toast de erro explicativo
- [ ] Preview da imagem selecionada antes de confirmar upload
- [ ] Botão "Confirmar foto" executa o fluxo de 3 etapas (presigned URL → S3 → PATCH profile)
- [ ] Botão "Cancelar" descarta a seleção
- [ ] Toast de sucesso ao concluir upload
- [ ] Toast de erro se qualquer etapa falhar

### AC5 — Exibição no sidebar
- [ ] `Layout.jsx` lê `avatar_url` do perfil via `useAuth()` ou query direta
- [ ] Se `avatar_url` preenchido: exibir `<img>` circular (40×40px, `object-fit: cover`, `border-radius: 50%`)
- [ ] Se `avatar_url` nulo: exibir placeholder com iniciais do `full_name` (mesma posição, mesmo tamanho)
- [ ] Avatar exibido no footer do sidebar, acima do e-mail e do botão "Sair"
