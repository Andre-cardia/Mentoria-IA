# Story 1.4 — Cadastro, Validação de Pagamento e Perfil do Usuário

## Status

Ready

## Story

**Como** comprador da Mentoria NeuralHub IA,
**Quero** criar minha conta na plataforma após realizar o pagamento,
**Para** que meu acesso seja vinculado ao meu pagamento real e meu perfil esteja completo.

## Critérios de Aceite

- [ ] AC-1: Comprador com e-mail validado no PagBank (status `PAID` na tabela `payments`) consegue criar conta em `/plataforma/registro`
- [ ] AC-2: Comprador sem pagamento confirmado vê mensagem de erro clara — não consegue criar conta
- [ ] AC-3: Na primeira vez que o usuário loga, é redirecionado para `/plataforma/completar-perfil` antes de acessar qualquer outra página
- [ ] AC-4: Página de completar perfil solicita: nome completo, telefone, como ficou sabendo da mentoria (campo opcional)
- [ ] AC-5: Após completar perfil, usuário é redirecionado para `/plataforma/modulos` e não volta para completar-perfil nas sessões seguintes
- [ ] AC-6: Tabela `profiles` populada corretamente com `user_id` referenciando `auth.users`
- [ ] AC-7: Webhook do PagBank continua funcionando sem interrupção — retorna 200 e salva email do comprador na tabela `payments`
- [ ] AC-8: Emails de compradores existentes recuperados do `raw_event` e populados na coluna `email` de `payments`

## Escopo

### IN
- Migration: `ADD COLUMN email` em `payments` + criação da tabela `profiles`
- Extração de email no `handle-webhook.js` (aditivo, sem quebrar fluxo)
- Página `/plataforma/registro` — valida pagamento por email antes de criar conta Supabase Auth
- Página `/plataforma/completar-perfil` — formulário de perfil (nome, telefone, origem)
- Guard no router: redireciona para completar-perfil se `profiles` incompleto
- Script SQL para popular emails dos compradores existentes via `raw_event`

### OUT
- Envio de email automático de boas-vindas
- Integração com CRM
- Upload de foto de perfil
- Alteração de senha na plataforma

## Dependências

- Story 1.3 Done (plataforma em produção) ✓
- PagBank em produção com vendas reais ✓
- Acesso ao Supabase SQL Editor para executar migrations

## Regras de Negócio

- A validação de pagamento usa **email** como chave de relacionamento (não user_id do PagBank)
- Apenas pagamentos com `status = 'PAID'` liberam o registro
- O campo `email` em `payments` será preenchido pelo webhook nas novas compras e via UPDATE retroativo nas existentes
- A tabela `profiles` usa `user_id` (FK para `auth.users`) como PK — um perfil por usuário
- Admin não precisa passar pela validação de pagamento para criar conta

## Arquitetura de Deploy (sem quebrar produção)

```
Passo 1: Migration DB (ADD COLUMN + CREATE TABLE profiles) → sem código
Passo 2: handle-webhook extrai email → push → webhook protegido por try/catch + 200
Passo 3: checkout aceita email opcional → push → backward-compatible
Passo 4: páginas React (registro + completar-perfil + guard) → push → features novas
Passo 5: UPDATE retroativo no SQL Editor → recupera emails existentes
```

## Tabela profiles

```sql
CREATE TABLE profiles (
  user_id    uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  full_name  text NOT NULL,
  phone      text,
  origin     text,           -- como ficou sabendo da mentoria
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now()
);

ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;

-- Usuário lê e edita apenas o próprio perfil
CREATE POLICY "perfil: usuario gerencia o proprio"
  ON profiles FOR ALL
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

-- Admin lê todos os perfis
CREATE POLICY "perfil: admin le todos"
  ON profiles FOR SELECT
  USING ((auth.jwt() -> 'user_metadata' ->> 'role') = 'admin');
```
