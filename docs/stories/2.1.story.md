# Story 2.1 — Edição Inline + Modal Confirmação + Toast

## Status

Ready for Review

## Story

**Como** administrador da plataforma,
**Quero** editar aulas inline e receber feedback visual de todas as minhas ações,
**Para que** eu possa gerenciar conteúdo de forma rápida e sem fluxos destrutivos.

## Contexto

Arquivo alvo: [AdminModulosPage.jsx](../../src/plataforma/pages/admin/AdminModulosPage.jsx)

**Problemas atuais resolvidos nesta story:**
1. Form de edição de módulo aparece no **topo da página** (fora do contexto do card) → mover para **inline no card**
2. Aulas **não possuem edição** — só exclusão, forçando delete + recriar → adicionar **edição inline**
3. `confirm()` nativo do browser para exclusões → substituir por **modal de confirmação**
4. Nenhuma operação tem feedback visual → adicionar **toast notifications**

## Acceptance Criteria

### AC1 — Form de Módulo Inline
- [x] O form de criar/editar módulo NÃO aparece mais no topo da página
- [x] Clicar em "Editar" num módulo expande um form **dentro do card do módulo**
- [x] Clicar em "+ Novo Módulo" abre um card novo no topo da lista (não um form global)
- [x] Form tem campos: Título (required), Descrição (optional)
- [x] Campo "Ordem" fica oculto (será gerenciado por drag-and-drop na Story 2.2)
- [x] Botões "Salvar" e "Cancelar" visíveis no form inline

### AC2 — Edição Inline de Aulas
- [x] Cada aula na lista tem um botão "Editar" (além do "Excluir" já existente)
- [x] Clicar em "Editar" transforma a linha da aula em modo de edição inline
- [x] Campos editáveis inline: Título, URL do vídeo, Duração (segundos)
- [x] Botões "Salvar" e "Cancelar" aparecem na linha em modo edição
- [x] Salvar chama `UPDATE` no Supabase e fecha o modo edição
- [x] Cancelar descarta as alterações sem salvar
- [x] Apenas uma aula pode estar em modo edição por vez

### AC3 — Modal de Confirmação para Exclusões
- [x] Excluir módulo abre modal com texto: "Excluir **{nome}** e todas as suas aulas? Esta ação não pode ser desfeita."
- [x] Excluir aula abre modal com texto: "Excluir a aula **{nome}**?"
- [x] Modal tem botão "Cancelar" (outline) e "Excluir" (vermelho)
- [x] `confirm()` nativo não é usado em nenhum lugar
- [x] Modal é um componente reutilizável `ConfirmModal`

### AC4 — Toast Notifications
- [x] Verificar se `sonner` já está no `package.json`; se não, instalar
- [x] Toast de sucesso verde após: criar módulo, editar módulo, criar aula, editar aula
- [x] Toast de sucesso após exclusão (módulo ou aula)
- [x] Toast de erro vermelho se qualquer operação Supabase falhar
- [x] Toasts duram 3 segundos e são posicionados no bottom-right
