# Story 8.3 — Blog: Admin — Editor de post

## Status

Ready

## Story

**Como** admin,
**Quero** criar e editar posts com editor rich text, inserir imagens (upload ou URL), definir cover, tags, SEO, status e agendamento,
**Para que** eu publique conteúdo rico e bem estruturado no blog.

## Contexto

Story central do Epic 8. Cria `AdminBlogEditorPage` com editor Tiptap v2. Serve para criar (`/novo`) e editar (`/:id/editar`) — mesmo componente, diferenciado pelo parâmetro de rota. Inclui todas as ferramentas editoriais.

## Acceptance Criteria

### AC1 — Instalação de dependências
- [ ] `npm install @tiptap/react @tiptap/starter-kit @tiptap/extension-image @tiptap/extension-link @tiptap/extension-placeholder react-helmet-async slugify`
- [ ] Dependências adicionadas ao `package.json`

### AC2 — Editor Tiptap
- [ ] Editor inicializado com `StarterKit` + `Image` + `Link` + `Placeholder`
- [ ] Toolbar com: negrito, itálico, h1, h2, h3, lista ordenada, lista não-ordenada, link, imagem, blockquote, separador
- [ ] Conteúdo armazenado como JSON (`editor.getJSON()`)
- [ ] Placeholder: "Escreva seu artigo aqui..."

### AC3 — Imagens no editor (modal com duas abas)
- [ ] Botão "Imagem" na toolbar abre modal
- [ ] Modal tem duas abas: "Upload" e "URL externa"
- [ ] **Aba Upload**: input de arquivo (JPEG/PNG/WebP, máx 5 MB) → POST `/api/blog/image-upload` com Bearer token → recebe `{ imageUrl }` → insere nó `<img>` no editor
- [ ] **Aba URL externa**: input de texto → valida que começa com `http` → insere nó `<img>` no editor
- [ ] Estado de loading no botão de confirmar durante upload

### AC4 — Cover image
- [ ] Área de upload de cover acima do editor
- [ ] Upload via POST `/api/blog/image-upload` (mesmo endpoint, prefixo S3 `blog-covers/`)
- [ ] Preview da imagem após upload
- [ ] Botão para remover cover

### AC5 — Campos do post
- [ ] Campo "Título" (obrigatório)
- [ ] Campo "Slug" (auto-gerado do título via `slugify`, editável manualmente)
- [ ] Select "Status": Draft, Publicado, Agendado, Arquivado
- [ ] Select "Visibilidade": Público, Privado
- [ ] Campo "Data de publicação" (datetime-local) — visível quando status=Agendado ou status=Publicado
- [ ] Select/input multi "Tags" — lista de tags existentes + opção de criar nova tag inline
- [ ] Campo "SEO Title" (opcional, max 60 chars com contador)
- [ ] Campo "SEO Description" (opcional, max 160 chars com contador)

### AC6 — Salvar e carregar
- [ ] Botão "Salvar rascunho" → upsert em `posts` com `status=draft`
- [ ] Botão "Publicar" → upsert com `status=published` e `published_at=now()`
- [ ] Modo edição: carrega post existente pelo `id` da rota e preenche todos os campos
- [ ] Modo criação: campos em branco, `author_id` e `author_name` do usuário logado
- [ ] Toast de sucesso/erro via `sonner`
- [ ] Após salvar, permanece na página (não redireciona)

### AC7 — API `POST /api/blog/image-upload`
- [ ] Aceita `fileName` e `contentType` no body + Bearer token
- [ ] Gera presigned URL S3 com prefixo `blog-images/` (ou `blog-covers/` via parâmetro `type`)
- [ ] Retorna `{ uploadUrl, imageUrl }` onde `imageUrl` é a URL pública permanente
- [ ] Rota equivalente criada em `server/routes/blog.js` para dev local
- [ ] Registrado em `server/index.js`: `app.use('/api/blog', blogRoutes)`

### AC8 — Sem regressões
- [ ] Outras páginas admin inalteradas
- [ ] `npm test` passa (32/32)

### Referência de estrutura do editor
```jsx
import { useEditor, EditorContent } from '@tiptap/react';
import StarterKit from '@tiptap/starter-kit';
import Image from '@tiptap/extension-image';
import Link from '@tiptap/extension-link';
import Placeholder from '@tiptap/extension-placeholder';

const editor = useEditor({
  extensions: [StarterKit, Image, Link.configure({ openOnClick: false }), Placeholder.configure({ placeholder: 'Escreva seu artigo aqui...' })],
  content: post?.content_json ?? '',
});
```
